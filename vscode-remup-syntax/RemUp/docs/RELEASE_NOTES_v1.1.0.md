## 🚀 新增功能详解

### 1. 自动换行（类似 Markdown）

#### 问题
之前版本中，RemUp 文件不会自动换行，长文本需要横向滚动，阅读体验差。

#### 解决方案
v1.1.0 默认启用自动换行，文字到达编辑器边界时自动折行。

#### 效果对比

**之前 ❌**
```
这是一行非常长的文本内容，会一直向右延伸超出屏幕范围，
需要拖动横向滚动条才能看到完整内容，阅读体验很差...
```

**现在 ✅**
```
这是一行非常长的文本内容，会在
编辑器边界处自动换行，无需横向
滚动即可看到完整内容，阅读体验
与 Markdown 完全一致！
```

### 2. 列参考线

#### 作用
- 80 列参考线：传统标准，适合打印和阅读
- 120 列参考线：现代标准，适合屏幕显示
- 帮助控制行长，提升代码可读性

#### 显示效果
```
这是 80 列内的内容，长度合适 |
这是超过 80 列但仍然在 120 列内的内容 |
这是超过 120 列的内容，建议换行 |
```

### 3. 可视化配置界面

#### 之前
需要手动编辑 `.vscode/settings.json` 文件

#### 现在
1. 按 `Ctrl+,` 打开设置
2. 搜索 "RemUp"
3. 勾选/取消勾选选项
4. 实时生效

### 4. 可自定义配置

提供 4 个配置项，满足个性化需求：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `remup.enableAutoWrap` | `true` | 启用自动换行 |
| `remup.wordWrapColumn` | `120` | 换行列宽 |
| `remup.enableRulers` | `true` | 显示列参考线 |
| `remup.rulersColumns` | `[80, 120]` | 参考线位置 |

### 架构优化

#### 新增 extension.js 入口文件
- 实现配置管理逻辑
- 监听配置变化

- 添加 `contributes.configuration` 贡献点

- 优化配置结构

### 性能提升
- 按需激活：仅在打开 RemUp 文件时加载
- 智能更新：配置变化时仅更新必要设置
- 无感应用：后台自动应用，不影响编辑

## 🔄 升级指南

### 从 v1.0.0 升级到 v1.1.0

#### 方式一：VSCode 内升级
1. 打开 VSCode
2. 按 `Ctrl+Shift+X` 打开扩展面板
3. 找到 "RemUp"
4. 点击 "更新"

#### 方式二：手动安装
1. 下载最新版 `.vsix` 文件
2. 在 VSCode 中按 `Ctrl+Shift+P`
3. 输入 "Extensions: Install from VSIX"
4. 选择下载的 `.vsix` 文件

### 兼容性

✅ **完全向后兼容**  
✅ 所有 v1.0.0 的功能继续有效  
✅ 现有配置文件不会被覆盖  
✅ 可选择使用或禁用新功能
