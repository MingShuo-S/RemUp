# RemUp 双语法速查表

## 📋 一分钟上手指南

### 传统语法 vs 极简语法

```
┌─────────────┬──────────────────┬──────────────────┐
│   功能      │   传统语法       │   极简语法       │
├─────────────┼──────────────────┼──────────────────┤
│  归档定义   │ --<名称>--       │ # 名称           │
│  卡片开始   │ <+主题           │ ## 主题          │
│  区域划分   │ ---区域名        │ ### 区域名       │
│  重要标签   │ (!: 内容)        │ [@重要]          │
│  参考链接   │ (>: #目标)       │ [@参考:#目标]    │
│  注卡批注   │ `内容 `[批注]     │ [内容 | 批注]     │
│  行内解释   │ >>解释           │ ^解释            │
└─────────────┴──────────────────┴──────────────────┘
```

---

## 🎯 快速示例

### 传统语法（完整形式）

```remup
--<学习笔记>--
<+Python 基础
(!: 重点) (>: #进阶)

---变量
`变量`[存储数据的容器] >>编程概念

---函数
def greet(name): >>定义函数
    return f"Hello, {name}"
/+>
```

### 极简语法（Markdown 风格）

```remup
# 学习笔记

## Python 基础
[@重点] [@参考:#进阶]

### 变量
[变量 | 存储数据的容器] ^编程概念

### 函数
def greet(name): ^定义函数
    return f"Hello, {name}"
```

## ✨ 核心特性

### 1️⃣ 结构化组织

```remup
# 归档名          ← 大主题分组
## 卡片主题       ← 知识点
### 区域名        ← 知识点的不同方面
```

### 2️⃣ 智能标签

```remup
[@重要]          ← 标记为重点
[@参考:#其他]    ← 链接到其他卡片
[@问题]          ← 标记为问题
[@信息]          ← 提供额外信息
```

### 3️⃣ 交互式批注

```remup
# 传统方式
`术语`[解释说明]

# 极简方式
[术语 | 解释说明]
```

### 4️⃣ 行内解释

```remup
# 传统方式
概念描述 >>简短解释

# 极简方式
概念描述 ^简短解释
```

## 🔧 Markdown 兼容

RemUp 完美支持常用 Markdown 语法：

```remup
**加粗文本**      → 粗体
*斜体文本*       → 斜体
==高亮文本==     → 黄色高亮
+放大文本 +      → 1.2 倍大小
++更大文本++     → 1.5 倍大小
`代码片段`       → 等宽字体
[链接文字](URL)   → 超链接
![图片描述](URL)  → 插入图片
```

## 💡 实战模板

### 词汇学习卡片

```remup
# 英语词汇

## vigilant
[@重要] [@参考:#例句]

### 释义
adj. 警惕的；警觉的 ^来自拉丁语

### 搭配
- be vigilant about [sth|某事] ^对...保持警惕
- remain vigilant ^保持警惕

### 例句
Stay vigilant [cyber threats|网络威胁] at all times. ^时刻保持警惕
```

### 概念理解卡片

```remup
# 编程概念

## 面向对象编程
[@核心概念] [@对比:#函数式编程]

### 定义
[OOP|面向对象编程] 是一种编程范式 ^基于对象而非动作

### 四大特性
- 封装 ^隐藏内部细节
- 继承 ^复用父类代码
- 多态 ^同一接口多种实现
- 抽象 ^提取共同特征

### 优势
- **易维护** ==代码结构清晰==
- **易扩展** ^支持增量开发
- **易协作** ^模块化分工
```

## 🎨 样式定制

### 标签符号含义

```
! = 重要         ? = 问题
> = 参考         < = 回链
i = 信息        ✓ = 已完成
☆ = 收藏        ▲ = 优先级
```

### 注卡归档

所有注卡会自动汇总到页面底部的"注卡归档"区域，方便复习。

## ⚡ 使用技巧

### ✅ DO（推荐）

- 保持文档内语法风格一致
- 使用标签建立知识关联
- 为专业术语添加注卡
- 用行内解释补充背景信息
- 合理使用区域划分内容

### ❌ DON'T（避免）

- 在同一卡片中混用两种语法
- 过度使用标签导致视觉混乱
- 注卡内容过长（建议<50 字）
- 忽略行内解释的重要性
