# RemUp v5.1 - 标签语法回归

## 🎯 重要变更

### 回归传统标签语法 `(符号：内容)`

经过实践验证，我们决定**回归到最初的标签模式**，移除 `[@标签]` 极简语法。

---

## ✨ 标签语法对比

### ❌ 已移除的极简标签

```remup
[@重要] [@参考:#目标] [@难度:⭐]
```

### ✅ 推荐的传统标签

```remup
(!: 重要) (参考:#目标) (难度:⭐)
```

---

## 📝 标签使用规范

### 基本格式

```
(符号：内容)
```

- **符号**：标签类型标识符（如 `!`, `>`, `*` 等）
- **内容**：标签的具体内容或引用

### 常用符号

| 符号 | 含义 | 示例 | 渲染样式 |
|-----|------|------|---------|
| `!` | 重要/基础 | `(!: 重要)` | 🔴 红色高亮 |
| `>` | 引用/链接 | `(>: #目标)` | 🔗 蓝色链接 |
| `*` | 星号/等级 | `(*: ⭐⭐⭐)` | ⭐ 星级显示 |
| 其他 | 默认样式 | `(备注：说明)` | ⚪ 灰色普通 |

### 多个标签

可以在一行中放置多个标签，用空格分隔：

```remup
## 卡片主题
(!: 重要) (参考:#前置) (难度:⭐)

### 区域名
内容...
```

## 🔧 技术实现

### lexer.py 改动

移除了 `label_md` 模式的处理，仅保留传统标签解析：

```python
def _process_inline_elements(self, line: str):
    """处理行内的各种元素 - 仅支持传统标签语法"""
    # 检查是否是标签（传统语法）
    label_match = self.PATTERNS['label'].match(line)
    if label_match:
        symbol = label_match.group(1).strip()
        content = [c.strip() for c in label_match.group(2).split(',')]
        self.tokens.append(('LABEL', f"{symbol}:{','.join(content)}", self.current_line_num))
        return
    
    # 处理普通行内容
    self._process_line_content(line)
```

### 正则表达式模式

```python
'label': re.compile(r'^\s*\(([^:]+):\s*([^)]+)\)')
```

匹配逻辑：
- `\(` - 左括号
- `([^:]+)` - 捕获符号（非冒号字符）
- `:\s*` - 冒号和可选空白
- `([^)]+)` - 捕获内容（非右括号字符）
- `\)` - 右括号

## ⚠️ 迁移指南

### 从极简标签迁移

如果你之前使用了 `[@标签]` 语法，需要转换为传统格式：

| 旧语法 | 新语法 | 说明 |
|-------|-------|------|
| `[@重要]` | `(!: 重要)` | 重要标签 |
| `[@参考:#目标]` | `(参考:#目标)` | 引用标签 |
| `[@难度:⭐]` | `(难度:⭐)` | 自定义标签 |
| `[@系列:#入门]` | `(系列:#入门)` | 系列标签 |

### 自动化转换

可以使用文本编辑器的查找替换功能：

1. 查找：`\[@([^\]]+)\]`
2. 替换：`($1)`
3. 手动调整符号（如添加 `!:` 等）

## 🎯 设计理由

### 为什么回归传统语法？

1. **语义更清晰**
   - `(符号：内容)` 结构明确
   - 符号系统便于快速识别标签类型

2. **视觉区分度好**
   - 圆括号 `()` 比方括号 `[]` 更醒目
   - 不会与注卡 `[内容][批注]` 混淆

3. **扩展性强**
   - 支持自定义符号系统
   - 易于添加新的标签样式

4. **用户习惯**
   - 老用户已经熟悉传统语法
   - 减少学习成本
