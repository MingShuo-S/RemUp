# 🎉 RemUp v4.0 - 双语法版本发布说明

## 📦 更新内容总览

RemUp v4.0 是一个**里程碑式更新**，在完全保留传统语法的基础上，新增了类似 Markdown 的极简语法支持。

---

## ✨ 核心特性

### 1️⃣ 双语法支持

现在你可以选择两种不同的语法风格：

#### 传统语法（经典）
```remup
--<归档名>--
<+卡片主题
(!: 重要标签)
---区域名
内容 `注卡`[批注] >>解释
/+>
```

#### 极简语法（新）
```remup
# 归档名

## 卡片主题
[@重要]

### 区域名
内容 [注卡 | 批注] ^解释
```

### 2️⃣ 完全兼容

- ✅ **100% 向后兼容** - 所有旧版文件无需修改
- ✅ **统一输出** - 两种语法生成相同的 HTML
- ✅ **自由切换** - 可在同一文档混用（不推荐）

### 3️⃣ 零学习成本

Markdown 用户可以立即上手，无需学习新符号。

## 🚀 快速开始

### 安装更新

```bash
git pull origin main
pip install -e .
```

### 创建第一个文件

#### 使用极简语法
```remup
# 我的笔记

## 简介
[@欢迎]

### 关于我
这是 [RemUp|记忆辅助标记语言] v4.0 ^全新双语法支持
```

#### 使用传统语法
```remup
--<我的笔记>--
<+简介
(i: 欢迎)
---关于我
这是 `RemUp`[记忆辅助标记语言] v4.0 >>全新双语法支持
/+>
```

### 编译文件

```bash
# 编译单个文件
remup build my_note.remup

# 指定主题
remup build my_note.remup -t DarkTheme

# 查看可用主题
remup --list-themes
```

## 📊 语法对照表

| 功能 | 传统语法 | 极简语法 |
|------|---------|---------|
| 归档定义 | `--<名>--` | `# 名` |
| 卡片开始 | `<+主题` | `## 主题` |
| 区域划分 | `---区域` | `### 区域` |
| 重要标签 | `(!: 内容)` | `[@重要]` |
| 参考链接 | `(>: #目标)` | `[@参考:#目标]` |
| 注卡批注 | `` `内容`[批注] `` | `[内容 \| 批注]` |
| 行内解释 | `>>解释` | `^解释` |

## 🎯 最佳实践

### ✅ DO（推荐）

- 保持文档内语法风格一致
- 为专业术语添加注卡
- 使用标签建立知识关联
- 用行内解释补充背景

### ❌ DON'T（避免）

- 在同一卡片中混用两种语法
- 过度使用标签导致混乱
- 注卡内容过长（<50 字为佳）
- 忽略行内解释的重要性
