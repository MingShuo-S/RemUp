## 🔄 转换规则

### 智能映射表

| [@] 中的关键词 | 映射符号 | 说明 |
|--------------|---------|------|
| 重要/基础/核心 | `!` | 红色高亮标签 |
| 参考/前置/后续/相关/对比/应用/示例 | `>` | 蓝色链接标签 |
| 系列/分类/难度/优先级/时间/状态 | 原文字 | 保留中文标识 |
| 重要性/高级特性/进阶 | `*` | 星级评价类 |
| 信息 | `i` | 信息提示 |
| 问题 | `?` | 问题标记 |
| 完成 | `✓` | 完成标记 |
| 其他 | `!` | 默认使用感叹号 |

## 🎯 转换后的优势

### 1. 语义更清晰

```remup
# 之前
[@重要] [@参考:#变量]

# 现在
(!: 重要) (>: #变量)
     ↑      ↑
   符号    符号
```

一眼就能看出：
- `!` - 重要警告
- `>` - 引用链接

### 2. 视觉区分度好

```remup
# 之前 - 都是 [@] 难以区分
[@重要] [@参考] [@问题]

# 现在 - 符号不同一目了然
(!: 重要) (>: 参考) (?: 问题)
 ↑       ↑       ↑
 !       >       ?
```

### 3. 样式更丰富

不同的符号对应不同的 CSS 样式：
- `!` → 红色高亮 🔴
- `>` → 蓝色链接 🔗
- `*` → 星级评价 ⭐
- `i` → 信息提示 ℹ️
- `?` → 问题标记 ❓
- `✓` → 完成标记 ✅

### 2. 特殊标签处理

部分标签的转换可能需要手动调整：

```remup
# 例如
[@重要性：⭐⭐⭐⭐⭐] → (!: 重要)  # 简化了
```

如果需要保留星级，可以手动修改为：
```remup
(*: ⭐⭐⭐⭐⭐)
```

### 3. 代码块内的标签

转换脚本会无差别替换所有 `[@...]`，包括代码块内的示例。这通常不是问题，因为代码块内的内容会被当作普通文本处理。
