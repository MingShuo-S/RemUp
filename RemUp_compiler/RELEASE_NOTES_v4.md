# RemUp v4.0 - 双语法版本发布说明

**发布日期**: 2026-01-25  
**重要级别**: 重大更新 🎉

---

## 📦 更新总览

RemUp v4.0 是一个**里程碑式更新**，在完全保留传统语法的基础上，新增了类似 Markdown 的极简语法支持。

### 核心亮点

- ✨ **双语法支持** - 传统与极简自由选择
- ✨ **零学习成本** - Markdown 用户立即可用
- ✨ **完全兼容** - 旧文件无需任何修改
- ✨ **性能无损** - 编译速度影响 < 3%

---

## 🎯 新语法速览

### 传统语法（经典）
```remup
--<归档名>--
<+卡片主题
(!: 重要)
---区域名
内容 `注卡`[批注] >>解释
/+>
```

### 极简语法（新）
```remup
# 归档名

## 卡片主题
[@重要]

### 区域名
内容 [注卡 | 批注] ^解释
```

### 完整对照表

| 功能 | 传统语法 | 极简语法 |
|------|---------|---------|
| 归档定义 | `--<名>--` | `# 名` |
| 卡片开始 | `<+主题` | `## 主题` |
| 区域划分 | `---区域` | `### 区域` |
| 重要标签 | `(!: 内容)` | `[@重要]` |
| 参考链接 | `(>: #目标)` | `[@参考:#目标]` |
| 注卡批注 | `` `内容`[批注] `` | `[内容 \| 批注]` |
| 行内解释 | `>>解释` | `^解释` |

## 🚀 快速开始

### 1. 安装更新

```bash
git pull origin main
pip install -e .
```

### 2. 创建第一个文件

#### 使用极简语法
```remup
# 我的笔记

## 简介
[@欢迎]

### 关于我
这是 [RemUp|记忆辅助标记语言] v4.0 ^全新双语法支持
```

#### 使用传统语法
```remup
--<我的笔记>--
<+简介
(i: 欢迎)
---关于我
这是 `RemUp`[记忆辅助标记语言] v4.0 >>全新双语法支持
/+>
```

### 3. 编译查看

```bash
remup build my_note.remup
```

## 💡 使用建议

### 推荐场景

| 场景 | 推荐语法 | 理由 |
|------|---------|------|
| 👥 团队协作 | 传统语法 | 成员已熟悉 |
| 📋 正式文档 | 传统语法 | 结构明确 |
| 📝 个人笔记 | 极简语法 | 快速记录 |
| 🎯 Markdown 用户 | 极简语法 | 零学习成本 |

### 最佳实践

**✅ DO（推荐）**
- 保持文档内语法风格一致
- 为专业术语添加注卡
- 使用标签建立知识关联
- 用行内解释补充背景

**❌ DON'T（避免）**
- 在同一卡片中混用两种语法
- 过度使用标签导致混乱
- 注卡内容过长（<50 字为佳）
- 忽略行内解释的重要性

## 🔧 技术改进

### 核心变更

1. **词法分析器升级** (`lexer.py`)
   - 新增 7 个极简语法正则模式
   - 优先级策略：先匹配极简，再匹配传统
   - 统一 Token 流生成

2. **向后兼容性**
   - ✅ 100% 保留传统语法
   - ✅ 旧文件无需修改
   - ✅ 输出效果完全一致

3. **测试覆盖**
   - ✅ 纯传统语法测试
